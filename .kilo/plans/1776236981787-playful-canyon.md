## Implementation Plan

### Phase 1: Database Setup (Manual Step Required)
**Action required from user:**
1. Log into Supabase dashboard
2. Open SQL Editor
3. Run contents of `SportCenter.Api/Database/supabase_schema.sql`
4. Verify tables created: `Locations`, `Events`, `EventSeries`, `EventLocations`

**Alternative for local development:**
- Use local PostgreSQL database (install if needed)
- Update `appsettings.json` connection string to local DB
- Run `dotnet ef database update` (migrations need to be re-created for PostgreSQL)

### Phase 2: Backend Connectivity
**Once DB is reachable:**
1. Verify connection string in `appsettings.json` is correct
2. Run backend: `dotnet run --project SportCenter.Api/SportCenter.Api.csproj`
3. Check logs for: `[DB] Migrations applied successfully.`
4. If migration fails due to no existing migrations for PostgreSQL, we need to:
   - Option A: Create new EF Core migration for PostgreSQL (`dotnet ef migrations add InitialCreate`)
   - Option B: Manually run SQL from `supabase_schema.sql` to create tables

### Phase 3: Frontend API Integration
**Replace in-memory service with real API calls:**

1. **Update `ActivityService.js`:**
   - Remove in-memory `activities` array
   - Implement `fetch` calls to `http://localhost:5000/api/events` (or backend URL)
   - Methods needed:
     - `getActivities(type)` → GET `/api/events` (filter by `IsDraft=false`)
     - `getDraftActivities()` → GET `/api/events/drafts`
     - `addActivity(activity)` → POST `/api/events` (with `IsDraft` flag)
     - `updateActivity(updated)` → POST `/api/events/{id}/publish` (for drafts)
     - `deleteActivity(id)` → DELETE `/api/events/{id}` (if needed)

2. **Handle CORS:**
   - Backend already allows `localhost:3000`
   - Ensure frontend calls use correct backend URL (could be `http://localhost:5000`)

3. **Test flow:**
   - Start backend (HTTPS typically `https://localhost:5001` or HTTP `http://localhost:5000`)
   - Start frontend: `npm start`
   - Create draft event → verify saved to DB
   - Open drafts tab → edit & publish → verify event updated in DB

### Phase 4: Optional - Recreate EF Core Migrations for PostgreSQL
Since we deleted the old SQL Server migrations, we should create new ones for PostgreSQL:

1. Ensure Npgsql provider is installed (already done)
2. Run: `dotnet ef migrations add InitialCreate` (from `SportCenter.Api` folder)
3. This will generate a migration that creates all tables with PostgreSQL syntax
4. The migration will be paired with the existing model definitions
5. Commit new migration to repo

**Note:** The SQL schema file is still useful for manual setup and documentation.
